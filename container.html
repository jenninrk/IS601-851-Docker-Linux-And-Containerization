What Is Containerization?

     Containerization by contrast very similar to virtualization in that computer resources are emulated, but it is different in that the resources that are emulated in virtualization are hardware in nature, and run on a single process, in the containerization environment the resources that are emulated are software in nature, specifically the operating system.

     As implemented, containerization does not require duplication of operating systems in a one to one relationship with each running application, as it does in the virtualized environment, on top of the host computer OS. Instead, a container “engine” is deployed on top of the host machine OS, to partition work at the application level and not the resource level.

     Further, containerization is more flexible, able to run on “bare metal” configurations, cloud instances, or even virtual machines. Additional benefits to containerization include faster deployments, greater throughput, and increased reliability.

     Two of the major container management software engines available in the marketplace today include Docker and Kubernetes. We will be working with these in future exercises.

     Some other benefits to containerization over virtual machines include multiple copies, scaling, Linux, and microservices. Containers typically deploy multiple copies of processes, and these multiple copies allow for rapid recovery from failure points. Scaling is easier also, as containerization is by design a much more streamlined process. If enhanced with smart web services software, it can self-edit, and more closely manage resources more tightly given fluid usage levels. Linux, the de facto operating system used in building and using containers, adds to system flexibility and ease of use, as command line capability renders containers easy to build and run, And finally, microservices assists in throughput reduction and performance enhancement, by reducing apps to smaller tasks performed through calls, rather than large processes requiring large resource allocation.

     Bottom line, containerization, like virtual machines, helps users achieve efficiencies and functionality without having to procure cost prohibitive and management intensive resources for one to one applications. But, unlike virtual machines, which can be overhead intensive on software necessary to resource management and applications handling, containerization also reduces overhead and throughput by minimizing software requirements to implement. The result is similar efficiencies to virtual machines, but with improved performance, stability, and functionality.
